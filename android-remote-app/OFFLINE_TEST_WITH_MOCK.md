# 无小车离线测试：使用模拟通信服务

## 一、用途

如果真实小车还没到，或者小车端通信服务暂时不可用，可以先用电脑启动一个模拟服务，验证安卓 App 是否能：

- 连接到 WebSocket 服务
- 发送 `/cmd_vel` 控制消息
- 按住按钮持续发送
- 松开按钮发送停止消息
- 点击急停发送停止消息

这个测试使用 App 的“模拟测试模式”，只能验证 App 的通信和控制消息发送，不代表真实小车一定会运动。

## 二、启动模拟服务

在电脑上打开命令行，进入：

```text
D:\D\dasanxia\carproject\app\tools
```

运行：

```text
start_mock_server.bat
```

如果不能运行 `.bat`，也可以直接运行：

```text
python mock_rosbridge_server.py
```

看到下面内容说明启动成功：

```text
[mock-tcp] mock control server listening on 0.0.0.0:9090
```

## 三、查看电脑 IP

在 Windows 命令行输入：

```text
ipconfig
```

找到当前 Wi-Fi 或热点对应的 IPv4 地址，例如：

```text
192.168.1.23
```

## 四、手机 App 连接

1. 如果使用 Android 模拟器，IP 输入 `10.0.2.2`。
2. 如果使用真实手机，手机和电脑连接同一个 Wi-Fi，并输入电脑 IPv4 地址。
3. 打开 `智能小车遥控`。
4. 确认按钮显示：

```text
模拟测试：开
```

5. IP 输入模拟器地址或电脑 IPv4 地址，例如模拟器使用：

```text
10.0.2.2
```

6. 端口输入：

```text
9090
```

7. 点击 `连接`。

如果连接成功，App 会显示已连接，方向按钮会变为可用。

## 五、验证控制消息

在手机上按住方向按钮，电脑命令行应该看到类似输出：

```text
[mock-tcp] received: {"op":"advertise","topic":"/cmd_vel","type":"geometry_msgs/msg/Twist"}
[mock-tcp] received: {"op":"publish","topic":"/cmd_vel","msg":{"linear":{"x":0.105,"y":0.000,"z":0.0},"angular":{"x":0.0,"y":0.0,"z":0.000}}}
```

松开按钮后，应该看到速度变成 0 的消息。

## 六、测试通过标准

| 测试项 | 通过标准 |
| --- | --- |
| App 连接模拟服务 | App 显示已连接 |
| 前进按钮 | 命令行收到 x 为正数的消息 |
| 后退按钮 | 命令行收到 x 为负数的消息 |
| 左右移动 | 命令行收到 y 不为 0 的消息 |
| 左右转 | 命令行收到 z 不为 0 的消息 |
| 松手停止 | 命令行收到 x/y/z 都为 0 的消息 |
| 急停 | 命令行收到 x/y/z 都为 0 的消息 |

## 七、常见问题

| 问题 | 处理 |
| --- | --- |
| 手机连不上电脑 | 确认手机和电脑在同一 Wi-Fi |
| 端口连接失败 | 确认模拟服务正在运行，端口是 9090 |
| Windows 防火墙拦截 | 允许 Python 通过防火墙 |
| IP 不对 | 用 `ipconfig` 重新查看 IPv4 地址 |
| 命令行没输出 | 确认 App 显示已连接，再按住方向按钮 |
