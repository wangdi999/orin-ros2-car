# 智能小车安卓遥控 App 测试清单

## 一、测试目标

验证成员 C 开发的安卓 App 能否在手机上运行，并完成连接小车、发送控制指令、调节速度和急停等基础功能。

## 二、测试前准备

| 检查项 | 是否完成 |
| --- | --- |
| Android Studio 能打开 `app` 工程 |  |
| 安卓手机已开启开发者模式 |  |
| 安卓手机已开启 USB 调试 |  |
| App 已成功安装到手机 |  |
| 手机和小车在同一网络 |  |
| 已确认小车 IP 地址 |  |
| 已确认小车端通信端口 |  |
| 小车底盘控制程序已启动 |  |
| 小车处于安全测试环境 |  |

## 三、测试步骤

### 1. App 启动测试

1. 在手机上打开 `智能小车遥控`。
2. 检查界面是否正常显示。
3. 检查 IP、端口、连接按钮、速度滑条、方向按钮、急停按钮是否存在。

预期结果：

```text
App 正常打开，界面无明显错位，未连接时方向按钮不可用。
```

### 2. 连接测试

1. 输入小车 IP。
2. 输入端口，默认使用 `9090`。
3. 点击 `连接`。
4. 观察连接状态文字。

预期结果：

```text
连接成功后显示“已连接”，方向按钮变为可用。
```

### 3. 速度调节测试

1. 将速度调到 20%。
2. 将速度调到 50%。
3. 观察界面速度文字是否变化。

预期结果：

```text
速度显示能跟随滑条变化。
```

### 4. 方向控制测试

第一次测试建议小车四轮悬空。

1. 按住 `前进`。
2. 松开按钮。
3. 按住 `后退`。
4. 松开按钮。
5. 按住 `左转`。
6. 松开按钮。
7. 按住 `右转`。
8. 松开按钮。

预期结果：

```text
按住按钮时小车运动，松开按钮后小车停止。
```

### 5. 急停测试

1. 按住任意方向按钮。
2. 点击或切换到 `急停`。
3. 观察小车是否停止。

预期结果：

```text
急停后小车立即停止。
```

### 6. 断开连接测试

1. 点击 `断开`。
2. 观察方向按钮状态。
3. 再次尝试点击方向按钮。

预期结果：

```text
断开后方向按钮不可用，不会继续发送运动指令。
```

## 六、异常处理

| 问题 | 可能原因 | 处理方式 |
| --- | --- | --- |
| App 连接失败 | IP 错误、端口错误、网络不通 | 检查手机和小车是否在同一网络 |
| 按钮无反应 | 未连接成功 | 先确认状态显示为已连接 |
| 小车不动 | 小车端控制程序未启动 | 检查小车底盘控制程序 |
| 方向相反 | 底盘方向定义不同 | 修改 App 中对应方向的速度正负号 |
| 小车速度过快 | 速度滑条过高 | 降低速度到 20% 到 30% |

## 七、无小车测试

如果小车暂时不可用，可以使用项目内的模拟通信服务进行离线测试：

```text
OFFLINE_TEST_WITH_MOCK.md
```

该测试可以先验证 App 是否能连接 WebSocket 服务并发送控制消息。
