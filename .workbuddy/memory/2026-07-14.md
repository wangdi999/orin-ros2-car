## 修复 CI 失败

两个 CI job 失败的原因：
1. `pytest.ini` 中的 `testpaths = tests/integration` 使所有 CI 的 `pytest -q` 被重定向到集成测试，而不是各自模块的本地测试。vision-python-lint 缺少 PyYAML，ros2-agent-ws 缺少 cv2 等依赖。
2. vision-python-lint CI 缺少 `opencv-python-headless`，新测试文件 test_visualizer/test_capture_manager 需要 cv2。

修复：
- `pytest.ini`: 移除 `testpaths = tests/integration`，保留 `pythonpath` 条目
- `.github/workflows/ci.yml`: vision-python-lint 添加 `opencv-python-headless PyYAML` 依赖；两个 job 的 `pytest -q` 改为 `pytest -q src/`（显式限定测试路径）
- `scripts/run_all_tests.ps1/.sh`: 同步改为 `pytest -q src/`

## 修复 CI ROS2 Foxy Build & Test 再次失败

根因：`test_safety_supervisor.py` 的 `from car_safety.safety_supervisor import monotonic_ms` 触发了 safety_supervisor.py 的完整导入链，其中 `from car_interfaces.msg import PatrolStatus` 依赖 `car_interfaces` 包。pytest 只将 `src/car_safety/` 加入 sys.path，不会自动加 `src/car_interfaces/`，导致 ModuleNotFoundError。

老测试 `test_arbiter.py` 没事是因为 `arbiter.py` 纯算法无 ROS2 依赖。

修复：
- `.github/workflows/ci.yml`: ros2-agent-ws job 在 colcon build 后添加 `source install/setup.bash`
- `test_safety_supervisor.py`: 移除 mock + 移除 `from car_safety.safety_supervisor import monotonic_ms`，改为内联 `monotonic_ms` 函数。仅保留 `from car_safety.arbiter import Velocity, Limits`（无 ROS2 依赖）
