## 1. 引言

### 1.1 编写目的

本文档为 Orin ROS2 智能巡检车系统的概要设计说明书，面向系统架构师、开发人员和后续运维人员。旨在阐明系统的整体架构、模块划分、技术选型、接口设计与部署方案，为详细设计、开发实现和系统集成提供依据。

### 1.2 项目背景

本项目基于 NVIDIA Jetson Orin 平台构建一套智能巡检车系统。用户通过 Web 控制台远程操控小车执行巡检任务，AI 视觉模块实时检测人员并进行异常行为（倒地等）识别，LangGraph Agent 编排层将自然语言指令转换为结构化巡检计划并执行。

### 1.3 设计目标

| 目标 | 描述 |
|------|------|
| **安全优先** | 急停独立于云端模型，速度仲裁、限幅和看门狗均在确定性层执行 |
| **人机协作** | 所有自动驾驶计划需人工审批后方可执行 |
| **模块解耦** | AI、控制、编排、底盘驱动各层独立部署和测试 |
| **离线可用** | 核心逻辑可在无 ROS2/GPU 环境下测试和验证 |
| **渐进式智能** | LLM 仅负责理解和规划，不直接控制执行器 |

### 1.4 术语与缩写

| 术语 | 说明 |
|------|------|
| ROS2 | Robot Operating System 2，机器人操作系统 |
| LangGraph | LLM 驱动的有状态工作流编排框架 |
| YOLOv8 | You Only Look Once v8，实时目标检测模型 |
| Nav2 | Navigation 2，ROS2 导航栈 |
| `cmd_vel` | 速度控制指令话题 |
| rosbridge | ROS2 与 WebSocket 之间的桥接协议 |
| Mecanum | 麦克纳姆轮，支持全向移动的底盘类型 |

## 4. 核心数据流

### 4.1 驾驶控制流

```
浏览器 (摇杆/键盘)
  │  POST /api/drive {forward, strafe, turn, linearLimit, angularLimit}
  ▼
control.mjs (buildTwistFromDriveInput)
  │  死区 → 归一化 → 钳位 → 限速 → geometry_msgs/Twist
  ▼
rosbridge.mjs (publishTwist)
  │  ws://car-ip:9090 → topic /cmd_vel
  ▼
Safety Supervisor (仲裁)
  │  急停 > 遥操作 > Nav2 导航 > 零速
  ▼
Mcnamu_driver_X3 (底盘驱动)
  │  串口协议 → 电机控制器
  ▼
小车运动
```

### 4.2 AI 检测 → 报警流

```
RGB-D 相机
  │  /camera/color/image_raw + /camera/depth/image_raw
  ▼
yolov8_inference (YOLOv8-TensorRT)
  │  人检测 → 深度关联 → 异常行为检测 → 消抖
  ├─ /chassis/ai_alarm (Alarm.msg)
  ├─ /camera/ai_detections (可视化 Image)
  ▼
ai_web_bridge (HTTP :6501)
  │  /video_feed (MJPEG)
  │  /api/alarms  (JSON)
  │  / (仪表盘页面)
```

### 4.3 Agent 巡检编排流

```
浏览器 (Agent 工作台)
  │  POST /api/v1/agent/requests {text: "巡检东门和停车场"}
  ▼
agent-runtime (LangGraph)
  │  classify_intent → load_context → generate_plan → validate_plan
  │
  │  ┌─── interrupt: request_approval ───┐
  │  │  等待人工 APPROVE/REJECT/EDIT      │
  │  └────────────────────────────────────┘
  │
  │  submit_task → Patrol Manager (ROS2 Service: /patrol/create + /patrol/control START)
  │
  │  ┌─── interrupt: wait_robot_event ───┐
  │  │  等待巡逻事件 (TASK_SUCCEEDED 等)   │
  │  └────────────────────────────────────┘
  │
  │  handle_robot_event → done → 生成报告
```

| 方法 | 路径 | 认证 | 功能 |

| GET | `/health` | 无 | 健康检查 |

## 6. 安全设计

### 6.1 多层防御体系

```
┌─────────────────────────────────────────────────────────────┐
│  第 5 层: 物理安全                                           │
│  - E-Stop 物理急停按钮（硬件电路）                              │
│  - 人工可随时取走/断电                                        │
├─────────────────────────────────────────────────────────────┤
│  第 4 层: 控制台看门狗                                        │
│  - 450ms 无驾驶指令 → 自动急停                                │
│  - WebSocket 断连 → 自动急停                                  │
│  - 页面 blur/unload → sendBeacon 急停                        │
│  - rosbridge 发送 + SSH 回退 双通道保障                        │
├─────────────────────────────────────────────────────────────┤
│  第 3 层: 安全仲裁器 (safety_supervisor)                      │
│  - 速度源优先级: EMERGENCY_STOP > MANUAL > NAV > ZERO         │
│  - 限幅: linear≤0.10, angular≤0.30 (可配)                    │
│  - NaN/Inf 拒绝                                              │
│  - 手柄/导航超时降级 (450ms/500ms)                             │
├─────────────────────────────────────────────────────────────┤
│  第 2 层: Agent 安全约束                                      │
│  - 计划必须人工审批 (LangGraph interrupt)                     │
│  - LLM 不生成坐标、速度、Topic、Shell                          │
│  - 仅允许预登记的地点 ID                                       │
│  - 禁用地点自动拒绝                                           │
├─────────────────────────────────────────────────────────────┤
│  第 1 层: 部署隔离                                            │
│  - gateway_mode 默认 mock，硬编码禁止实车模式                   │
│  - Agent 在独立进程运行，不直接访问 ROS2 DDS                   │
│  - 急停路径不依赖 LLM/网络                                    │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 急停路径

急停可由以下任一方式触发：

1. **用户操作**: 控制台急停按钮、空格键、Agent 急停 API
2. **自动保护**: 驾驶看门狗超时、WebSocket 断连、页面关闭
3. **安全仲裁器**: 急停信号锁定，持续发送零速度

所有路径最终汇聚到 `safety_supervisor`，通过 20Hz 定时器持续发布零 Twist。

## 8. 技术选型

| 层面 | 技术 | 选型理由 |
|------|------|----------|
| 前端 | React 19 + Vite 6 | 轻量，HMR 快速开发 |
| 后端 | Node.js (ESM) | 与前端统一语言，ws 库成熟 |
| SSH | PuTTY plink | Windows 环境 SSH 客户端 |
| ROS2 通信 | rosbridge (WebSocket) | 无需 ROS2 客户端库即可通信 |
| 编排 | LangGraph + checkpoint | 有状态工作流，支持中断/恢复 |
| LLM | OpenAI 兼容 API | 统一接口，支持 mock 模式 |
| AI 推理 | YOLOv8 + TensorRT FP16 | Jetson Orin GPU 加速 |
| 数据库 | SQLite (WAL) | 嵌入式，零配置 |
| 接口定义 | FastAPI + Pydantic | 类型安全，自动文档 |
| 测试 | pytest + Node.js --test | 轻量，无需额外框架 |

## 9. 约束与限制

1. **ROS2 版本**: 固定为 ROS2 Foxy，依赖小车 Docker 镜像 `icar/ros-foxy:1.0.2`
2. **Python 版本**: 通用模块 Python 3.8，agent-runtime 需要 Python 3.10+
3. **网络**: 小车与 Windows 开发机需在同一 Wi-Fi 网段（ohcar9）
4. **Nav2 地图**: 未创建真实地图前，`navigate_to_pose` 行为由仿真决定
5. **地点标定**: 未在真实地图上标定的地点保持 `enabled: false`
6. **Agent 网关**: 当前硬编码 `mock` 模式，未实现 rosbridge 模式的实车网关
